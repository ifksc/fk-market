## Стек

- **PHP 8.3** + **Laravel 11** (API + Sanctum)
- **MySQL 8** (InnoDB, utf8mb4)
- **Redis** (кэш + очереди)
- **nginx** (фронт-прокси)
- **Docker Compose** — локальная инфраструктура одной командой (и продакшн — тем же compose, с другим `.env`)

## Как запустить локально

```bash
cd "/Users/lavrov/Documents/Claude/Projects/FK.market/backend"
chmod +x setup.sh
./setup.sh
```

Первый запуск — 2-3 минуты. Скрипт сам:
1. Проверит Docker
2. Поднимет MySQL + Redis
3. Установит Laravel 11 в `src/` (если ещё не установлен)
4. Включит API-роутинг + Sanctum
5. Скопирует наши миграции/модели/сиды/роуты
6. Создаст `.env`, сгенерит `APP_KEY`
7. Запустит `php artisan migrate:fresh --seed`

## Полезные команды

### Dev (локально)

```bash
docker compose down              # остановить всё
docker compose down -v           # сбросить БД
docker compose logs -f app       # логи PHP
docker compose exec app sh       # войти в контейнер
docker compose exec app php artisan tinker   # REPL
docker compose exec app php artisan migrate:fresh --seed   # пересоздать БД
```

### Prod (сервер `194.87.118.214`)

```bash
# SSH-подключение (ключ настраивается на этапе Фаза 4)
ssh -p 22 deploy@194.87.118.214

# Развёртывание из git на сервере
cd /opt/fk.market && git pull && docker compose up -d --build
docker compose exec app php artisan migrate --force
docker compose exec app php artisan config:cache
```

> [!warning] Продакшн-данные
> На продакшне **не** запускать `migrate:fresh` и `migrate:fresh --seed` — это удалит все данные. Только `migrate` (идёт только вперёд).

## Ошибки и как лечить

| Симптом | Причина | Лечение |
|---------|---------|---------|
| `Docker is not running` | не запущен Docker Desktop | запустить, подождать |
| `Port 8000 already in use` | порт занят | `docker compose down` или сменить порт в compose |
| `SQLSTATE Access denied` | сломался `.env` | `docker compose down -v && ./setup.sh` |
| Migration SQL error | несовместимость версии MySQL | сообщить, поправлю миграцию |
