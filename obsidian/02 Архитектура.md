## Основные потоки

### Путь покупки

```
1. Покупатель → каталог → карточка товара
2. В корзину → оформление (email + параметры товара)
3. Выбор оплаты → редирект на FKwallet
4. FKwallet → вебхук «paid»
5. Laravel:
   - status: pending → paid
   - для каждой позиции → в зависимости от fulfillment_mode:
     • stock   → резерв stock_items + копия payload → delivered
     • api     → джоба в очередь → ProviderGateway::order() → delivered
     • manual  → fulfillment_tasks.queued → админ в очереди
6. Email с кодами + ЛК: «Готово»
```

### Ценообразование

```
price_final = price_base × (1 + effective_markup / 100)

effective_markup = product.markup_pct
  OR pricing_rules.scope=product
  OR pricing_rules.scope=seller
  OR pricing_rules.scope=category
  OR pricing_rules.scope=global
```

Пересчитывается джобой при изменении `price_base`, `markup_pct` или `pricing_rules`.

### Очередь выдачи

Сущность: [[Глоссарий#`fulfillment_tasks`|fulfillment_tasks]] (одна задача = одна позиция заказа).

```
Order created → OrderItem created
       ↓
   продукт.mode == ?
       ├── stock   → резерв из stock_items
       │              ├── есть остаток → immediate delivery
       │              └── нет остатка → fulfillment_tasks(manual)
       ├── api     → fulfillment_tasks(api) → job → ProviderGateway
       │              ├── success → delivered
       │              └── fail × retries → fulfillment_tasks(manual)
       └── manual  → fulfillment_tasks(manual) → админ
```

SLA — `deadline_at`, по умолчанию 4 часа на ручную задачу (`settings.order_sla_minutes_manual`).

| Служебные | `settings`, `audit_logs` |

### Ключевые архитектурные решения

- **`seller_id` с v1** — чтобы миграция в v2 не ломала БД. См. [[Журнал решений#2026-04-23]].
- **`fulfillment_mode` в `products`** — разные режимы на одном товаре невозможны (намеренно). Для сложных случаев — fallback.
- **Шифрование полей** — `stock_items.payload`, `order_items.delivered_payload`, `providers.credentials` через [[Глоссарий#Laravel `Crypt`|Laravel Crypt]].
- **Денормализация `price_final`** — для скорости каталога.
- **Pricing rules отдельно** — гибкая система наценок (global/category/seller/product) с приоритетами.

## Платёжный слой

Интерфейс `PaymentGateway` (PHP) — методы `createPayment()`, `verifyWebhook()`, `getStatus()`, `refund()`. Реализации:
- `FKWalletGateway` — в MVP
- `YooKassaGateway` / `RobokassaGateway` — fallback (в случае проблем)

## Слой поставщиков

Интерфейс [[Глоссарий#`ProviderGateway`|ProviderGateway]]:
- `listProducts($filters)` — импорт каталога
- `getStock($external_id)` — проверка остатка
- `order($external_id, $params)` — оформить заказ у поставщика
- `getStatus($provider_order_id)` — обновить статус

Реализации: `DigisellerGateway`, `KinguinGateway`, `G2GGateway`, `SteamBotGateway`.

## Безопасность

- `users.password` — `hashed` cast (argon2id/bcrypt)
- Sanctum токены для API
- Middleware `auth:sanctum` + role-check для админки
- CSRF на веб-формах, origin-check для SPA
- Rate limiting: 60 req/min на публичном API, 20 на авторизационных
- Crypt-шифрование чувствительных полей
- [[Глоссарий#Аудит-лог|Audit log]] админских действий
