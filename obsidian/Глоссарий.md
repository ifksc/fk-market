# Глоссарий FK.market

Термины, специфичные для проекта. Использовать в [[Журнал решений]], [[Kanban]] и коммуникации.

## Архитектура и выдача

### `fulfillment_mode`
Режим получения товара покупателем. Одно из: **`stock`** (готовый ключ со склада), **`api`** (запрос к API поставщика в момент оплаты), **`manual`** (задача в очередь админу). У каждого товара хранится в `products.fulfillment_mode`.

### `fulfillment_fallback`
Что делать, если основной режим упал. Сейчас: `manual` (падаем в ручную очередь) или `none` (возврат денег). Пример: `api → fallback manual` — если Kinguin не ответил, админ выдаёт сам.

### `stock_items`
Склад ключей / аккаунтов / кодов — по одной записи на продаваемую единицу. Поле `payload` зашифровано.

### `fulfillment_tasks`
Задача на выдачу. Одна задача = одна позиция заказа. Имеет SLA-дедлайн (`deadline_at`), статус, исполнителя (`assignee_id`).

### SLA
Service Level Agreement — договорное время на выдачу. В настройках `order_sla_minutes_manual = 240` минут (4 часа) для ручных задач.

## Поставщики

### `provider`
Внешний источник товаров. Примеры: **Digiseller** (подписки), **Kinguin** (ключи игр), **G2G** (игровые услуги), **Steam-бот** (свой, для скинов CS2/Dota2).

### `ProviderGateway`
Единый PHP-интерфейс в Laravel с методами `listProducts()`, `order()`, `getStatus()`. Каждая реализация — отдельный класс (DigisellerGateway, KinguinGateway и т. д.).

### `provider_products`
Маппинг «товар поставщика → товар FK.market». Позволяет импортировать каталог поставщика и подвязывать нужные позиции к нашему каталогу.

## Платежи

### `FKwallet`
Платёжный провайдер (Free-Kassa). Приём платежей (карта/СБП/крипта/кошелёк) и выплаты (payout для продавцов в v2).

### Режимы платежа
- `fkwallet` — обычная оплата через FKwallet-редирект
- `fkwallet_wallet` — оплата с кошелька FK (внутренний баланс пользователя)
- `balance` — оплата с внутреннего баланса FK.market
- `manual` — зафиксирован вручную админом (налик / фактура)

### Вебхук
Колбэк от FKwallet об успешной/неуспешной оплате. Подпись HMAC проверяется на бэке обязательно.

## Ценообразование

### `price_base`
Закупочная цена / себестоимость.

### `markup_pct`
Индивидуальная наценка на товар. Если `null` — применяются [[#pricing_rules]].

### `price_final`
Итоговая витринная цена (`price_base × (1 + markup_pct/100)`). Денормализованное поле, пересчитывается при изменении base/markup/rules.

### `pricing_rules`
Правила наценок. Поле `scope`: `global` / `category` / `seller` / `product`. При конфликте побеждает **более специфичное правило** и/или большее значение `priority`.

## Пользователи и роли

### `customer`
Обычный покупатель.

### `admin`
Оператор платформы. Управляет товарами, заказами, очередью выдачи.

### `seller`
Продавец (актуально с v2). В v1 существует один **системный продавец** `platform`.

### `moderator`
Роль для v2: модерирует товары и продавцов перед публикацией.

## Заказы

### `public_number`
Номер заказа для покупателя (`FK-2026-08421`). Отличается от `id` — безопасен для показа.

### Статусы заказа
`pending` → `paid` → `fulfilling` → `completed`. Плюс ветви: `failed`, `refunded`, `cancelled`.

## Безопасность

### Laravel `Crypt`
Модуль шифрования Laravel. Использует AES-256-CBC + HMAC, ключ — `APP_KEY` из `.env`. Применяется к `stock_items.payload`, `order_items.delivered_payload`, `providers.credentials`.

### `AsEncryptedString`
Cast для Eloquent, автоматически шифрует/расшифровывает поле при записи/чтении.

### Аудит-лог
Таблица `audit_logs` с действиями админов (write-only). Нужна для расследований.
